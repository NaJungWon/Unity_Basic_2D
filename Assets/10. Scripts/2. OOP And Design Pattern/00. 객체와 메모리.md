# 객체와 메모리
> 객체가 메모리에 어떻게 저장되고, 값타입과
> 참조타입이 어떻게 다른지 알아 봅시다.

# 객체란?
상태(데이터 == 필드)와 행동(기능 = 메서드)를 하나로 묶은 단위.
클래스(설계도)로 찍어낸 실물(인스턴스)이다.

객체가 없으면 관련 데이터가 흩어져, 메모리에도 제각각 놓입니다.
```csharp
string enemyName = "고블린";
int enemyHp = 100;
int enemyAtk = 5;

string goblinName = "고블린";
int goblinHp = 100;
int goblinAtk = 5;
string slimeName = "슬라임";
int slimeHp = 100;
int slimeAtk = 5;

string goblinName = "고블린";
string slimeName = "슬라임";
int goblinHp = 100;
int slimeHp = 100;
int goblinAtk = 5;
int slimeAtk = 5;
```

## 개념
### 클래스, 구조체와 객체
```csharp
class Monster
{
    public string Name;
    public int Hp;
    public int Atk;
}
// new = 인스턴스(객체) 하나 생성
Monster a = new Monster();
```
- 클래스, 구조체 = 설계도
- 'new'로 만든것 = 인스턴스(객체)
- 설계도 하나로 여러 객체를 찍어 낼 수 있음


### 값 타입 vs 참조 타입(구조체 vs 클래스)

|    | 값 타입(value)                        | 참조 타입(reference)                   |
|----|------------------------------------|------------------------------------|
| 예시 | `int`,`float`,`bool`,`enum`,`struct` | `class`,`array`,`string`,`delegate` |
| 변수(공간)에 든 것 | 값 그 자체 | 주소(참조) |
| 대입/전달 시 | 독립적인 복사본 | 주소 복사 -> 같은 객체를 공유한다 |
| 저장 위치 | 보통 스택, 힙 메모리에 저장된 레퍼런스 타입 객체 내부 | 실체는 힙(변수에는 주소) |

```csharp
// 값 타입 : 독립적으로 복사됨
int x = 10;
int y = x; // 값 복사. x 데이터의 10값을 복사하여 y 데이터에 대입함
y = 99; // y에 99를 대입하더라도 x는 여전히 10

// 참조 타입 : 주소가 복사되어 "같은 객체"를 공유
Monster a = new Monster {Hp = 100};
Monster b = a; // 주소 복사. a와 b는 같은 몬스터를 가리킴
b.Hp = 1; // a.Hp도 1이 됨
```

함수 인자(매개변수)도 같습니다. **값 타입은 복사본**이 넘어가서 원본이 변하지 않고,
**참조 타입은 주소**가 넘어가서 원본이 바뀔 수 있습니다.

## 메모리 - 스택과 힘
프로그램이 사용하는 주요 메모리 공간들 중
프로그래머가 런타임에 다룰 수 있는 공간은
스택, 힙 두 공간입니다.

- 스택(stack) : 함수의 지역 변수,값 타입이 쌓이는 빠른 임시 공간. 함수가 끝나면 자동으로 정리됨.
- 힙(heap) : `new` 로 만든 참조 타입 객체가 저장되는 넓은 공간. GC(가비지 컬렉터)가 관리.

`Monster a = new Monster();`를 실행하면
- 실제 데이터(Name, Hp, Atk)는 힙에 저장되고
- 변수 'a'는 그 힙 객체의 주소만 저장한다.
