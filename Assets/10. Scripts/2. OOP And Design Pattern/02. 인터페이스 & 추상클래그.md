# 인터페이스 & 추상클래스
> "무엇을 할 수 있는지"만 약속하는 **인터페이스**와
> 일부는 구현하고 일부는 자식에게 강제하는 **추상클래스**.
> 거의 모든 디자인 패턴의 재료.
 
## 1. 인터페이스와 추상클래스란?
### **인터페이스(interface)**
구현 없이 **"할 수 있는 일의 목록"** 만 정한 약속(계약). 
이 인터페이스를 상속받은 클래스는 해당 목록의 함수를 
**반드시** 구현해야 한다.
> 자격증. "운전을 할 수 있다"는 능력만 보증한다.
> 트럭이든 택시든 상관 없다. => Can-do

### 추상 클래스(abstract class)
일부 기능은 미리 구현해두고,
나머지는 자식에게 강제로 넘기는 미완성 부모.
단독으로 `new`키워드를 통해 인스턴스화 될 수 없다.
> 미완성 설계도. 자동차 뼈대(엔진, 바퀴)는 그려져 있고,
> 좌석수 같은 세부사항은 자식이 채운다. => Is-A

### 요약
> 객체의 정체성과 관련되어있다면 **추상클래스**,
> 약속된 능력/기능 등을 의미하면 **인터페이스**


## 2. 상황 
전투에서 데미지를 처리하는 코드. 공통 부모가 없으면
대상 타입마다 갈래가 생긴다.

```csharp

public void Attack(object target)
{
    if(target is Player p) p.TakeDamage(10); 
    else if(target is Enemy e) e.TakeDamage(10);
    if(target is Boss b) b.TakeDamage(10);
}
```
> 공통 약속이 없어서. 호출하는 쪽이 모든 구체적인 타입을 알아야 한다.
> 즉, 의존성이 생긴다( = 강한 결합도)를 갖고 있다.
> 새로운 전투 대상(타입)이 추가될 때 마다 if문이 늘어 난다.
 
## 3. 적용
### 3.1 추상 클래스 사용
> 공통은 물려주고, 핵심은 강제한다.

```csharp
public abstract class CombatEntity : MonoBehaviour
{
    // abstract : 몸체가 없다. 
    //"데미지를 입는 방식은 자식이 알아서 구현"하라고 강제
    public abstract void TakeDamage(int damage);
    
    // virtual + 빈 몸체 : 기본 구현을 주되, 
    // 필요하면 자식이 덮어쓸 수 있다. (선택 구현)
    public virtual void TakeHeal(int heal);
}
```
- **abstract** : 구현 강제(컴파일 에러)
- **virtual** : 선택 구현(기본 구현 제공, 덮어쓰기 허용)

위의 추상클래스로 각 클래스(Player, Enemy, Boss)의
정체성을 묶어주면 아래처럼 활용 가능

```csharp
public void Attack(CombatEntity target)
{
    target.TakeDamage(10); // 다형성이 알아서 올바른 구현을 호출함.
}
```

### 3.2 인터페이스 활용
```csharp
public interface IHpObserver
{
    void OnHpChanged(int currentHp, int maxHp);
    // 구현은 없음. 오직 약속만
}

public class HpBarUI : MonoBehaviour, IHpObserver
{
    public void OnHpChanged(int currentHp, int maxHp)
    {
        // 체력바 갱신 로직
    }
}

public class HpTextUI : MonoBehaviour, IHpObserver
{
    public void OnHpChanged(int currentHp, int maxHp)
    {
        // 텍스트 변경 처리
    }
}
```
한 클래스는 부모를 **하나만** 상속하지만, 인터페이스는 **여러개** 상속할 수 있다.

## 3.3 정리

|    | 추상클래스            | 인터페이스               |
|----|------------------|---------------------|
| 의미 | ~이다(Is-A, 정체성)   | ~할 수 있다(Can-Do, 능력) |
| 필드 보유 | 가능 | 불가(상수/프로퍼티만) |
| 기본 구현 | 줄 수 있음 (virtual) | 원칙상 없음 |
| 다중 구현 | 부모는 하나만 | 여러개 가능 |
