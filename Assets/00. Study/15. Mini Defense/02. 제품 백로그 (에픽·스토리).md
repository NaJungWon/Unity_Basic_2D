# 미니 디펜스 — 제품 백로그 (에픽 · 스토리)

> 유저가 체험할 수 있는 기능 단위로 나눈 백로그. 각 행을 그대로 보드 카드로 옮겨 사용한다.
> **완료 조건(Definition of Done)** 을 만족하면 그 스토리는 Done.
> 기획 근거: [01. 컨셉 기획서](<01. 컨셉 기획서.md>) · [02. 시스템 기획서](<02. 시스템 기획서.md>)

---

## 에픽 목록

| ID | 에픽 | 설명 |
|---|---|---|
| E1 | 맵 & 경로 | 그리드, 타일, 적 이동 경로 |
| E2 | 유저 인터페이스 | 조작, 정보 UI, HUD |
| E3 | 유닛 배치, 관리 | 소환, 업그레이드, 판매 기능 |
| E4 | 웨이브 & 적 | 호출, 스폰, 진군, 죽음, 종료 |
| E5 | 전투 | 조준, 발사, 피해, 처치 |
| E6 | 게임 흐름 & 게임 종료 | 게임 상태 전이. 승리, 패배 처리 |
| E7 | 유저 데이터, 저장 & 불러오기 | 경제, 게임 진행상황, 진행 상태 보존, 복원 |
| E8 | 부팅(초기화) & 게임 데이터 | 외부 데이터 로드, 각종 시스템 초기화 |

## 유저 스토리 백로그

| ID | 에픽 ID | 유저 스토리 | 완료 조건(Definition of Done) |
|---|---|---|---|
| 1 | E1 | 타일 그리드의 출력을 확인할 수 있다. | 유니티 Scene에 게임 설정에서 지정된 크기의 그리드가 생성된다 |
| 2 | E1 | 적이 생성되는 스폰 타일을 구분할 수 있다. | 출력된 그리드에서 지정된 스폰 타일을 시각적(초록색)으로 구분할 수 있다 |
| 3 | E1 | 적이 도착하는 목표 타일을 구분할 수 있다. | 출력된 그리드에서 지정된 목표 타일을 시각적(빨강색)으로 구분할 수 있다 |
| 4 | E1 | 적이 진군할 경로 타일을 구분할 수 있다. | 출력된 그리드에서 지정된 경로 타일을 시각적(주황색)으로 구분할 수 있다 |
| 5 | E1 | 화면의 타일 부분을 클릭하여 해당 타일의 좌표를 확인할 수 있다. | 화면 입력으로 타일을 클릭 시, 올바른 좌표를 로그로 출력한다 |
| 6 | E2 | 빈 타일을 클릭하면 유닛 생성 메뉴가 출력된다 | 3종의 유닛 및 각각의 가격이 표시되는 생성 메뉴가 출력되며, 자원이 부족하다면 해당 항목이 비활성화 상태로 출력된다 |
| 7 | E2 | 유닛 생성 메뉴에서 유닛을 선택하고 소환 버튼을 눌러 소환을 진행할 수 있다. | 전투 가능한 개체가 생성되며, 플레이어의 자원이 차감된다 |
| 8 | E2 | 아군 유닛을 클릭하면 관리(업그레이드/판매) 메뉴가 출력된다 | 현재 레벨, 비용, 스탯 등의 정보가 표시된다 |
| 9 | E2 | 아군 유닛을 클릭하면 사거리가 표시된다 | 선택된 유닛의 공격 사거리가 원형으로 표시된다 |
| 10 | E2 | 관리 메뉴를 이용하여 특정 유닛을 업그레이드할 수 있다 | 비용 차감, 스탯 상승, 최대 레벨 상한, 업그레이드된 수치 확인 가능 |
| 11 | E2 | 관리 메뉴를 이용하여 특정 유닛을 판매할 수 있다 | 유닛을 판매하여 일부 자원을 환불받고, 타일이 비어 있는 상태로 바뀐다 |
| 12 | E2 | [다음 웨이브] 버튼을 이용하여 웨이브를 시작할 수 있다 | 준비 → 진행으로 게임 진행 상태 변경. 진행 중에는 버튼 비활성 |
| 13 | E2 | 플레이어로서 보유 광물이 화면에 보이고 변할 때 갱신된다 | 획득/소비 즉시 반영 |
| 14 | E2 | 플레이어로서 남은 체력이 화면에 보이고 갱신된다 | 누수 시 즉시 감소 표시 |
| 15 | E2 | 플레이어로서 현재/전체 웨이브가 화면에 보인다 | 예: 1/3 표시 |
| 16 | E3 | 입력을 받아 특정 타일에 유닛을 소환할 수 있다 | 배치 가능한 타일에 한해 유닛이 생성되고 해당 타일이 점유 상태가 된다 |
| 17 | E3 | 입력을 받아 특정 타일의 유닛을 제거할 수 있다. | 해당 타일의 유닛이 제거되고 타일 점유가 해제된다 |
| 18 | E3 | 입력을 받아 특정 타일을 점유 중인 유닛의 정보를 확인할 수 있다. | 해당 유닛의 종류·레벨·스탯을 조회할 수 있다 |
| 19 | E4 | 웨이브 데이터의 구성대로 적이 스폰된다 | 그룹·수·간격·지연이 데이터대로 반영된다 |
| 20 | E4 | 적이 경로를 따라 목표로 이동한다 | 웨이포인트를 순서대로 따라 이동한다 |
| 21 | E4 | 적이 목표에 도달하면 적이 사라지며, 플레이어의 체력이 감소된다 | 적별 누수 피해(LeakDamage)만큼 체력 감소 |
| 22 | E4 | 웨이브의 적을 모두 처리하면 클리어 보상을 받고 준비 단계로 돌아간다 | 스폰 완료 & 생존 0 판정 후 클리어 보상 지급 |
| 23 | E5 | 유닛이 사거리 안의 적을 조준 방식대로 고른다 | 선두/최근접 등 선택 결과 정확 |
| 24 | E5 | 유닛이 조준 대상에게 투사체를 발사한다 | 풀에서 재사용; 대상으로 이동 |
| 25 | E5 | 명중 시 크리·방어·저항이 계산되어 체력이 깎인다 | 공식대로 최종 피해; 이벤트 버스 경유 |
| 26 | E5 | 적이 죽으면 사라지고 광물을 획득한다 | 처치 보상 지급; 소멸 |
| 27 | E5 | 포격병이 착탄 반경의 여러 적을 동시에 타격한다 | SplashRadius·MaxTargets 반영 |
| 28 | E5 | 크리티컬이 터지면 화면에 표시된다 | 크리 시 피드백(색/수치) |
| 29 | E5 | 지휘관 주변 공격 유닛의 공격력이 강화된다 | 범위 진입 시 증가·이탈 시 원복; 지휘관 제외 |
| 30 | E5 | 버프가 공격 속도에도 적용된다 | AttackInterval 감소 반영 |
| 31 | E6 | 게임이 상태(준비/진행/승리/패배)에 따라 동작한다 | 상태별 저장/배치 가능 규칙 적용 |
| 32 | E6 | 마지막 웨이브를 클리어하면 승리 화면이 출력된다 | 승리 전이·표시 |
| 33 | E6 | 체력이 0이 되면 패배 화면이 출력된다 | 패배 전이·표시 |
| 34 | E7 | 웨이브 사이에서 현재 진행을 저장한다 | 진행 중 저장 잠금; 파일 생성 |
| 35 | E7 | 저장을 불러와 이어서 플레이한다 | 유닛 재배치·광물·체력·웨이브 복원 |
| 36 | E8 | 게임을 시작하면 6종 테이블이 읽혀 런타임 데이터로 조립된다 | 콘솔에 로드 요약 출력; 실패 시 명확한 에러 |
| 37 | E8 | 데이터 로드 후 시작 광물·기본 체력이 설정되고 준비 상태로 진입한다 | HUD에 시작값(광물 300·체력 20) 반영 |
